## 🏁 چک‌لیست نهایی استقرار (Deployment Checklist)

برای اینکه مطمئن شوید همه مراحل را به درستی انجام داده‌اید، از جدول زیر استفاده کنید. هر مرحله‌ای که انجام دادید را تیک بزنید:

| مرحله | شرح عملیات | روی Master (مغز) | روی Worker (کارگر) | وضعیت |
| :--- | :--- | :---: | :---: | :---: |
| **فاز ۱** | تنظیم Hostname، خاموش کردن Swap و تنظیمات کرنل | ✅ | ✅ | [ ] |
| **فاز ۲** | نصب `containerd` و ست کردن Mirrorهای تحریم‌شکن | ✅ | ✅ | [ ] |
| **فاز ۳** | دانلود باینری‌ها (`kubeadm`, `kubelet`, ...) و ساخت سرویس | ✅ | ✅ | [ ] |
| **فاز ۴** | اجرای `kubeadm init` و تنظیم دسترسی `kubectl` | ✅ | ❌ | [ ] |
| **فاز ۵** | نصب شبکه داخلی (Calico) | ✅ | ❌ | [ ] |
| **فاز ۶** | اجرای دستور `kubeadm join` برای اتصال به کلاستر | ❌ | ✅ | [ ] |
| **تست** | بررسی وضعیت نودها با دستور `kubectl get nodes` | ✅ | ❌ | [ ] |

**💡 نکته کاربردی:** اگر در هر یک از مراحل Worker به مشکل خوردید، ابتدا مطمئن شوید که **فاز ۱ تا ۳** دقیقاً مشابه سرور Master اجرا شده باشند.

---

### 🛠️ Final cluster check

اگر بعد از نصب نیاز به دستورات سریع داشتید، این‌ها کاربردی‌ترین‌ها هستند:

*   **مشاهده وضعیت نودها:** `kubectl get nodes`
*   **مشاهده وضعیت تمام پادها:** `kubectl get pods -A`
*   **مشاهده لاگ‌های kubelet:** `journalctl -u kubelet -f`
*   **ساخت توکن جدید برای Join:** `kubeadm token create --print-join-command` (فقط روی Master)
