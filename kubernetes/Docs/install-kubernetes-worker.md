## 👷 بخش ویژه: راهنمای نصب روی نودهای Worker
*اگر شما روی سرور Worker هستید، نیازی نیست تمام مراحل Master را طی کنید. فقط این مراحل را به ترتیب انجام دهید:*

### گام ۱: آماده‌سازی (مشابه Master)
این مراحل را روی هر Worker اجرا کنید:
1. **تنظیم اسم:** `sudo hostnamectl set-hostname k8s-worker-1`
2. **آپدیت سیستم:** `sudo apt update && sudo apt upgrade -y`
3. **غیرفعال کردن Swap:** 
   ```bash
   sudo swapoff -a
   sudo sed -i '/swap/s/^/#/' /etc/fstab
   ```
4. **تنظیمات شبکه کرنل:** (همان دستورات فاز ۱-بند ۴ داک اصلی را اجرا کنید تا `br_netfilter` و `ip_forward` فعال شوند).

### گام ۲: نصب موتور اجرایی (containerd)
دقیقاً مراحل **فاز ۲** داک اصلی (نصب containerd و تنظیم Mirrorهای تحریم‌شکن) را روی Worker اجرا کنید. 
**نکته:** بدون تنظیم Mirrorها، نود Worker نمی‌تواند ایمیج‌های برنامه‌های شما را دانلود کند و پادها در حالت `ImagePullBackOff` می‌مانند.

### گام ۳: نصب باینری‌های کوبرنتیز
دقیقاً مراحل **فاز ۳** داک اصلی (دانلود مستقیم `kubeadm`, `kubelet`, `kubectl`, `kube-proxy` و انتقال به `/usr/bin`) را اجرا کنید. 
**فراموش نکنید:** حتماً فایل سرویس `kubelet.service` را بسازید و آن را `enable` کنید (همانطور که در فاز ۳ توضیح داده شد).

### گام ۴: اتصال به کلاستر (The Join)
حالا نوبت به مرحله نهایی می‌رسد. دستور `join` را که از سرور Master (در فاز ۴) دریافت کردید، اینجا اجرا کنید:

```bash
sudo kubeadm join <IP-MASTER>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>
```

## 🧐 سوالات متداول برای Workerها (Troubleshooting)

**۱. دستور Join را گم کردم، چه کنم؟**
ترس نکنید! روی سرور **Master** دستور زیر را بزنید تا یک توکن جدید برای شما بسازد:
```bash
kubeadm token create --print-join-command
```
خروجی این دستور همان دستور `join` است که باید روی Worker اجرا کنید.

**۲. چطور بفهمم Worker من به درستی وصل شده؟**
باید به سرور **Master** بروید و دستور زیر را بزنید:
```bash
kubectl get nodes
```
اگر نام نود Worker را در لیست دیدید و وضعیت آن **Ready** بود، یعنی همه چیز عالی است.

**۳. اگر وضعیت نود NotReady بود چه کنیم؟**
معمولاً این اتفاق به دو دلیل می‌افتد:
- یا **CNI (Calico)** روی Master نصب نشده است (فاز ۵ را چک کنید).
- یا **containerd** روی Worker درست تنظیم نشده یا سرویس `kubelet` فعال نیست.
